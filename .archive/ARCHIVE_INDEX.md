# Legacy Documentation Index

## Purpose

The `legacy/` directory contains historical documentation that has been superseded or is no longer actively maintained.

**Current documentation:** See `core/` and `reference/` directories.

---

## Structure

### `legacy/bot-specific/`
EditorBot application-specific documentation (not infrastructure):
- DOCUMENTATION_INDEX.md — Old documentation index
- LEARNING_PATH.md — Bot learning guide
- WORKFLOW_GUIDE.md — Bot workflow details
- VISUAL_GUIDE.md — Bot diagrams
- STATE_MACHINE.md — Bot state machine
- QUICK_START.md — Bot quick start

**Status:** Retained for historical reference. Not relevant to infrastructure pipeline.

### `legacy/sessions/`
Session logs and temporal notes:
- SESSION_2026-01-20.md — Documentation consolidation session

**Status:** Historical record of work sessions.

### `legacy/fixes/`
Historical fix summaries:
- FIX_SUMMARY.md — Initial bot fixes
- TRANSCRIPTION_UPDATE.md — Whisper integration

**Status:** Current state documented in core/IMPLEMENTATION.md.

### `legacy/` (root level)
- INTEGRATION_AUDIT.md — Student repo integration notes
- VERIFICATION.md — Testing procedures

**Status:** Historical context; may contain useful verification patterns.

---

## Usage

**For current information:** Use documentation in `core/` and `reference/`.

**For historical context:** Browse `legacy/` to understand how the project evolved.

**Do not rely on legacy docs for operational procedures.**
